# Instruktion till Codex

Du ska bygga en responsiv HTML-upplevelse utifrån filerna i projektmappen.

Läs filerna i den här ordningen innan du börjar bygga:

1. 00_FOUNDATIONS.md
2. 01_SOMMARLASNING.md
3. 03_CREATIVE_BRIEF.md
4. 02_ARKITEKTENS_ANTECKNINGSBOK.md

Bygg inte en PowerPoint på webben.

Bygg ett digitalt resonemang.

Läsaren ska steg för steg ledas genom en lugn, mänsklig och affärsnära berättelse.

# Viktigaste budskapet

Att ge Holmens bästa människor bättre förutsättningar att göra det de redan är bäst på.

# Leverans

Skapa en komplett statisk webbupplevelse.

Skapa följande struktur:

/presentation

- index.html
- styles.css
- script.js
- assets/ (vid behov)

Upplevelsen ska kunna publiceras som en statisk webbsida via exempelvis GitHub Pages eller Netlify.

Inga serverberoenden ska krävas.

All kod ska vara enkel, ren och möjlig att vidareutveckla.

# Publicering och delning

Målet är att kunna dela upplevelsen via en enkel webblänk.

Läsaren ska inte behöva:
- installera något
- ladda ner filer
- förstå tekniken bakom

Upplevelsen ska kännas som att någon har byggt ett personligt digitalt magasin.

Inte som en publicerad presentation.


# Upplevelse

Det ska kännas som:

- modern digital artikel
- lågmäld storytelling
- skog
- långsiktighet
- mänsklighet
- kvalitet
- mycket luft

Det ska inte kännas som:

- AI-demo
- startup
- konsultpresentation
- intern PowerPoint
- kampanjsida

# Struktur i HTML

Bygg upplevelsen i dessa sektioner:

1. Start
   - Titel: Mer tid för det som betyder mest
   - Kärnmening: Att ge Holmens bästa människor bättre förutsättningar att göra det de redan är bäst på.

2. Kundresan
   - Visa att kundresan är grunden.
   - Lyft budskapet: Kundresan har inte blivit mindre relevant. Möjligheten att agera på den har blivit större.

3. Observationerna
   - Byggstenarna finns redan
   - Den bästa virkesköparen visar redan vägen
   - Den största potentialen finns mellan affärerna
   - En starkare relation är också en starkare affär

4. Från administration till relation
   - Visualisera enkel kontrast:
     Idag: mer administration, mindre relation
     Imorgon: mindre administration, mer relation

5. Teknikens roll
   - Budskap: Tekniken ska bära relationen. Inte tvärtom.
   - AI, data och digitala verktyg ska visas som stöd, inte huvudperson.

6. Förmågan
   - Förmågan att stärka relationen med privata skogsägare
   - Visa lärloopen:
     Lyssna → Förstå → Prioritera → Utveckla → Följa upp → Lära

7. Beslutsmodellen
   - Varje beslut ska belysas ur perspektiven:
     - Den privata skogsägaren
     - Virkesköparen
     - Affären
     - Digital utveckling och data

8. Om tre år
   - Virkesköparen säger: "Jag får lägga mer tid på det jag faktiskt är bäst på."
   - Skogsägaren säger: "Holmen hör av sig när det verkligen betyder något."
   - Holmen säger: "Vi använder teknik för att förstärka det mänskliga."

9. Avslutning
   - Vi ska inte ersätta det mänskliga med teknik.
   - Vi ska använda teknik för att förstärka det mänskliga.

# Design

Använd:

- ljus bakgrund
- naturliga jordnära toner
- mörkgrön eller skogsgrön som accent
- mycket whitespace
- stora rubriker
- lugna övergångar
- enkla linjer
- enkla visuella modeller i CSS/SVG

Använd inte:

- robotikoner
- AI-hjärnor
- neon
- starka gradienter
- stockfoto
- överdrivna animationer

# Interaktion

Använd subtila scroll-animationer om det passar.

Animationer ska vara lugna och inte störa läsningen.

Om animation och läsbarhet krockar: välj läsbarhet.

# Språk

Använd svenska.

Använd Holmens språk:

- privata skogsägare
- virkesköpare
- relation
- skoglig kompetens
- stärka
- bygga vidare
- förmåga

Undvik:

- kundupplevelse
- digital transformation
- AI-first
- disrupt
- kundinsikter som huvudbegrepp

# Viktigt

Det här ska inte kännas som ett färdigt förslag.

Det ska kännas som ett genomtänkt resonemang som bjuder in till samtal.

När läsaren är klar ska hon tänka:

"Det här vill jag prata mer om."
